## Quick start

### View the dashboard

Open in a browser (no server required):

```bash
open dashboard/live_seller_churn_dashboard_20260618.html
```

### Refresh data from CSV

```bash
python scripts/build_canvas.py
```

This reads `data/raw/streams_first_stream_2026.csv` and writes `data/processed/canvas_data.json`.

To update the live dashboard, re-embed the JSON into `dashboard/live_seller_churn_dashboard_20260618.html` (`const DATA = …` block), then reopen the file.

### Validate logic (optional)

```bash
cd tools && node harness.js
```

## Data files

### Input: `data/raw/streams_first_stream_2026.csv`

One row per **live stream event** for sellers who first streamed on Live in 2026.

| Column | Used for |
|--------|----------|
| `SLR_ID` | Seller ID (group streams per seller) |
| `ID` | Stream / event ID |
| `STRT_TM` | Stream start time |
| `RETAIL_WEEK` | Retail week for this stream row |
| `first_stream_dt` | Seller’s first-ever Live stream date |
| `SELLER_COUNTRY` | Country filter |
| `FCSD_VRTCL_SUP_GRP_revised` | Sub category filter |

### Processed: `data/processed/canvas_data.json`

Built by `scripts/build_canvas.py`. Contains seller list with `(retail_week, vertical)` pairs plus country/tier/org/vertical lookup arrays.

### Dashboard: `dashboard/live_seller_churn_dashboard_20260618.html`

Self-contained HTML with embedded `DATA` JSON and all churn calculations in JavaScript (`compute()`).

## Seller cohorts (2026)

- **Cohort** = retail week of a seller’s **first Live stream** in 2026 (`26RW01` … `26RW24`).
- Each seller belongs to **exactly one cohort** (minimum `RETAIL_WEEK` per `SLR_ID`).
- Cohorts **21–24** are hidden in the dashboard (insufficient follow-up for fair comparison).
- ~2,570 sellers in the current extract; data through retail week **24**.

---

## Churn definitions

All churn metrics are computed in **`compute()`** inside the dashboard HTML (not in Python).

### Summary cards (weighted averages)

`weighted % = sum(churned) / sum(starters)` across eligible cohorts.

| Card | At | Churned if no stream in |
|------|-----|------------------------|
| **New seller** | W4 | W1, W2, W3 |
| **Growing seller** | W10 | W7, W8, W9 |
| **Mature seller** | W16 | W13, W14, W15 |

Eligibility: cohort must have enough weeks of follow-up (`maxWeek − cohort ≥ 4, 10, or 16`).

### Heatmap table (from W4)

At column **Wk** (`k ≥ 4`): churned if no stream in the prior 3 tenure weeks (`W(k−3)`, `W(k−2)`, `W(k−1)`).

### Tenure weeks

If cohort retail week is `c`, tenure week **Wk** = retail week `c + k`.  
W4 ≈ 1 month, W10 ≈ 2.5 months, W16 ≈ 4 months since first stream.

## Dashboard components

- **Summary cards** — total sellers; new / growing / mature weighted churn
- **Filters** — country (Other Countries hidden from dropdown); sub category
- **Heatmap** — 3-week gap churn from W4; toggle Churn % / Counts
- **Line chart** — per-cohort churn at W4, W10, W16

---

## Filters and edge cases

- **Country:** excludes sellers not matching selection; empty = all countries.
- **Sub category:** only streams in selected verticals count toward week sets; can affect cohort in edge cases.
- **Comebacks:** streaming again removes churn for weeks where the look-back window includes that stream.
- **Excluded cohorts:** 26RW21–24 not shown (`EXCLUDED_COHORTS` in dashboard JS).

---

## Retail week calendar (2026)

Sunday–Saturday weeks; 26RW01 = Jan 4–10, 2026. Full `RW_DATES` map lives in the dashboard HTML.

## Quick reference formulas

```
NEW SELLER (W4):   no stream in weeks c+1, c+2, c+3
GROWING (W10):     no stream in weeks c+7, c+8, c+9
MATURE (W16):      no stream in weeks c+13, c+14, c+15
TABLE at Wk (k≥4): no stream in weeks c+k-3, c+k-2, c+k-1

c = cohort retail week (first stream week)
```
